----------------------------------------------------------------
  PART 2: WEB EXPORT SIZES (resize to these before uploading)
----------------------------------------------------------------

These are your export targets for the website.
Always resize down from your master file — never up.


  [ COMIC PAGES (portrait — any series) ]

    Portrait single page (US comic, manga, etc.):
      Export width:     900 px
      Height:           auto (maintain aspect ratio)
      Format:           WebP (preferred) or JPEG
      Quality:          85%
      Expected size:    80–250 KB for B&W ink art

    2-page spread (landscape):
      Export width:     1400 px
      Height:           auto
      Format:           WebP / JPEG 85%
      Expected size:    150–350 KB


  [ COMIC STRIPS (horizontal) ]

    Daily strip, 3-panel mini, or any image wider than 2:1:
      Export width:     1200 px
      Height:           auto (will be short — that's correct)
      Format:           WebP / JPEG 85%
      Expected size:    60–180 KB

    NOTE: The website reader auto-detects strips (width:height > 2:1)
    and switches to horizontal scroll mode on mobile automatically.
    No extra setup needed — just export at 1200px wide and upload.


  [ SITE ASSETS ]

    Series cover (homepage card):
      Size:             600 × 800 px  (3:4)
      Format:           WebP / JPEG 85%
      Filename:         cover.jpg  (in the series folder)

    Series header / title treatment:
      Max width:        500 px
      Format:           PNG  (keep transparency for drop shadow effect)
      Filename:         header.png  (in the series folder)

    Series / chapter background — COVER mode (default):
      Width:            1920 px
      Height:           ~1080 px (16:9 — or taller for portrait screens)
      Format:           WebP / JPEG 80%
      Filename:         bg.jpg  (in series or chapter folder)
      series.json:      "backgroundMode": "cover"  (or omit — cover is default)

      Scales to fill the entire screen behind the content panel.
      Good for: character illustrations, atmosphere shots, splash art,
      any single image you want to bleed edge to edge.

    Series / chapter background — TILE mode:
      Size:             200–800 px square  (seamless, any square works)
      Format:           WebP / JPEG / PNG
      Filename:         bg.jpg  (same filename — mode set in series.json)
      series.json:      "backgroundMode": "tile"

      Repeats in all directions to fill the screen.
      The tile scrolls with the page (not fixed).
      Good for: textures, graph paper, dot grids, crosshatch,
      halftone patterns, newsprint, lined paper, wood grain,
      brick, any repeating motif.

      Tile design tips:
        — The left edge must match the right edge seamlessly.
          The top edge must match the bottom edge seamlessly.
        — Design at 2× the final size, then scale down
          (e.g. design at 600×600, export at 300×300).
        — Smaller tiles (200–300 px) feel denser and more textured.
          Larger tiles (600–800 px) give more breathing room between repeats.
        — Photoshop: Filter → Other → Offset (wrap) to check seams.
          Clip Studio: use the Seamless Pattern canvas type.
        — File size matters less — a 30 KB seamless PNG tiling
          looks the same as a 500 KB cover image, but loads 10× faster.

      Per-chapter override:
        Drop a different bg.jpg in the chapter folder AND add
        "backgroundMode": "tile" (or "cover") to that chapter.json
        to use a different background style for just that chapter.

    Site logo (header bar):
      Height:           72 px  (displays at 36px, 2× for Retina)
      Width:            auto
      Format:           SVG (preferred — scales perfectly at any size)
                        PNG (fallback — use if SVG isn't available)
      Filename:         assets/logo.svg  (what the site actually uses)

    Artist photo (About page):
      Size:             400 × 400 px minimum  (displays as 140px circle)
      Format:           PNG, JPEG, or WebP
      Filename:         assets/artist.png  ← what about.html expects


  [ FAVICON & APP ICONS ]

    Drop all of these in the assets/ folder.
    All browsers and platforms are covered by this set.

    File                           Size        Notes
    ─────────────────────────────────────────────────────────────
    favicon.ico                    16+32 px    Multi-size ICO.
                                               Legacy browsers,
                                               bookmark bars.
                                               Required.

    favicon-16x16.png              16 × 16     Tabs, small spots.

    favicon-32x32.png              32 × 32     Standard browser tab.
                                               Most common use.

    apple-touch-icon.png           180 × 180   iOS "Add to Home Screen"
                                               icon. No rounded corners —
                                               iOS does that automatically.

    android-chrome-192x192.png     192 × 192   Android home screen icon.

    android-chrome-512x512.png     512 × 512   Android splash / high-DPI.
                                               Best quality source.

    site.webmanifest               —           Already created. Do not edit
                                               unless you change the site name.

    TOTAL files to make: 6 images + the webmanifest (already done).

    Design tips:
      - Design at 512 × 512, then scale down — never up.
      - Simple bold shapes. At 16px detail is invisible.
      - The duck head, a grinning face, initials, or the logo mark
        all work well. Keep it recognizable at tiny size.
      - Dark background (#0e0e0e) or transparent both work.
      - Transparent PNG is fine for all except the ICO.

    How to generate the full set from one image:

      Easiest:  https://favicon.io  or  https://realfavicongenerator.net
        Upload your 512×512 PNG → download the full set → drop in assets/.

      Manual (Photoshop / GIMP):
        Export each PNG at the sizes above.
        For .ico: use IcoFX (free), GIMP's ICO export,
        or convert via https://cloudconvert.com/png-to-ico
        Include both 16×16 and 32×32 layers in the ICO file.

    Quick test after uploading:
      Open the site in Chrome → open DevTools (F12) →
      Application tab → Manifest — should show your icons listed.


----------------------------------------------------------------
  PART 3: STRIP MODE THRESHOLDS (how the reader detects format)
----------------------------------------------------------------

The reader measures each image's width ÷ height automatically.
Based on that ratio it applies the right display mode:

  Ratio           Display mode
  ─────────────────────────────────────────────────────
  < 1.0           Portrait  — normal vertical fit, full width
  1.0 – 2.0       Landscape — normal fit, full width (spreads, Sunday halves)
  > 2.0           STRIP     — horizontal scroll on mobile, fixed height

  Common sizes and their ratios:
    US comic single page        0.66:1   → Portrait
    Manga B5 single page        0.71:1   → Portrait
    US comic 2-page spread      1.31:1   → Landscape (normal)
    Sunday half-page strip      1.40:1   → Landscape (normal)
    3-panel Sunday mini strip   3.00:1   → STRIP ✓
    Daily strip (tabloid)       7.00:1   → STRIP ✓
    Daily strip (broadsheet)    7.20:1   → STRIP ✓
    IAGL (approx)               3.5:1    → STRIP ✓

  You do not need to tag or configure anything.
  Just export at the right width and the reader handles it.


----------------------------------------------------------------
  PART 4: NAMING CONVENTION
----------------------------------------------------------------

Always use zero-padded numbers so files sort correctly:

  CORRECT:   01.jpg  02.jpg  09.jpg  10.jpg  11.jpg
  WRONG:     1.jpg   2.jpg   9.jpg   10.jpg  (sorts as 1,10,11,2...)

  Cover:     cover.jpg
  Header:    header.png
  Background: bg.jpg


----------------------------------------------------------------
  PART 5: CUSTOM CURSORS (per-series)
----------------------------------------------------------------

Each series can have a TWO-FILE cursor set that activates on the
series page and reader. Drop files in the series assets folder
and scan.js picks them up automatically.

  Two files, two states:

    cursor.png   Idle state — shown whenever cursor is on the page
    cursor.gif   Hover state — animates when cursor is over any
                 link, button, or other interactive element

  Supply one or both. If you only have a PNG it shows everywhere
  as a static cursor. If you only have a GIF it's used for both
  states. Both together gives you the idle → animate transition.

  How it works (no CSS cursor tricks):

    Browsers don't animate GIFs in CSS cursor: url() — they fall
    back silently to a plain arrow. The site uses a fake cursor:
    a <div> that follows the mouse via JS. background-image DOES
    animate GIFs. The native cursor is hidden with cursor: none.

  Recommended canvas size:

    32 × 32 px    Standard — works everywhere, sharp.
    64 × 64 px    HiDPI / Retina — more visible on sharp screens.

    Stick to 32×32 or 64×64. Larger sizes aren't reliably
    supported and the cursor div is 32×32 (scales with contain).

  Format notes:

    PNG (cursor.png — idle state):
      Export as PNG-24 or PNG-32 with alpha channel.
      Design on a transparent canvas. No background fill.
      The outline/shape IS the cursor.

    GIF (cursor.gif — animated hover state):
      256 colours only — keep the palette simple.
      Set to loop infinitely (loop count = 0 in GIF settings).
      Keep under 8–12 frames for best performance.
      Binary transparency only — hard edges, no anti-aliasing
      on transparent edges (GIF doesn't support partial alpha).

  Hotspot (click point):

    The hotspot is always the top-left corner (0, 0).
    Design with the active point at the top-left:
      Pen / quill   → nib tip at top-left
      Hand / finger → fingertip at top-left
      Arrow         → arrow tip at top-left

  Design tips:

    - Keep it simple and readable at 32×32.
    - High-contrast outline or 1px drop shadow helps it show
      on both light and dark series backgrounds.
    - Suggested themes:
        IAGL   — retro quill pen, smiley face, small character
        Melvin — something goofy (his face, a rubber duck, a sock)
        Dio    — candle, skull, crescent moon, clawed hand

  Folder placement (always in the series assets/ subfolder):

    comics/iagl/assets/cursor.png
    comics/iagl/assets/cursor.gif
    comics/melvin/assets/cursor.png
    comics/melvin/assets/cursor.gif
    comics/dio/assets/cursor.png     ← done ✓
    comics/dio/assets/cursor.gif     ← done ✓

  Site-wide cursors (fallback when no series is active):

    assets/cursor.png
    assets/cursor.gif

    These are always shown on non-series pages (homepage, blog,
    about, archive) and as the default before a series loads.

  Run node scan.js after adding any cursor file.
  No other code changes needed.


----------------------------------------------------------------
  PART 6: RECOMMENDED TOOLS
----------------------------------------------------------------

  Squoosh (free, browser):    https://squoosh.app
    Drag and drop. Convert to WebP, compare quality live.
    Great for one-off images.

  XnConvert (free, desktop):  https://www.xnview.com/en/xnconvert/
    Batch resize + convert an entire folder.
    Best for converting a full chapter at once.

  Photoshop:
    File → Export → Export As → choose WebP or JPEG, set quality 85

  GIMP (free):
    File → Export As → choose .webp

  Clip Studio Paint:
    File → Export (Single Layer) → choose format and resize
